# Kalibr Configuration Files

This directory contains configuration files for Kalibr IMU-camera calibration (Phase 1).

## Files

### Input Configurations (Edit These)

1. **`bluelily_imu.yaml`** - IMU noise parameters
   - Pre-configured with MPU6500 typical values
   - Kalibr will refine these during calibration
   - No changes needed unless using different IMU

2. **`kinect2_cam.yaml`** - Camera intrinsics and settings
   - ⚠️ **MUST UPDATE** `intrinsics` with your Kinect's actual values
   - Run: `./scripts/extract_kinect_intrinsics.sh` to get your values
   - Replace the placeholder values in this file

3. **`target.yaml`** - Checkerboard target specifications
   - ⚠️ **MUST MEASURE** your printed checkerboard exactly
   - Use calipers to measure corner-to-corner spacing
   - 1mm error = bad calibration

### Output Files (Generated by Kalibr)

After running Kalibr, you'll get:

- **`camchain-imucam-<timestamp>.yaml`** - ⭐ CRITICAL OUTPUT
  - Contains T_cam_imu transform
  - Use in ORB-SLAM3 config (Phase 2)
  - Convert to TF2 with: `./scripts/kalibr_to_tf2.py`

- **`imu-<timestamp>.yaml`** - Refined IMU parameters
  - Better noise estimates than initial values
  - Use in ORB-SLAM3 config (Phase 2)

- **`report-imucam-<timestamp>.pdf`** - Calibration quality report
  - Check reprojection error (<1.0 pixel is good)
  - Review residual plots
  - Verify gravity magnitude ≈ 9.81 m/s²

- **`results-imucam-<timestamp>.txt`** - Text summary

## Quick Start

### 1. Extract Kinect Intrinsics
```bash
# Launch Kinect bridge
ros2 launch kinect2_ros2_cuda kinect2_bridge.launch.py

# Extract intrinsics
./scripts/extract_kinect_intrinsics.sh

# Update kinect2_cam.yaml with the extracted values
nano kalibr_configs/kinect2_cam.yaml
```

### 2. Measure Checkerboard
```bash
# Print checkerboard from: https://github.com/ethz-asl/kalibr/wiki/downloads
# Mount on rigid board
# Measure corner-to-corner spacing with calipers

# Update target.yaml
nano kalibr_configs/target.yaml
```

### 3. Record Calibration Bag
```bash
# Launch both bridges
ros2 launch kinect2_ros2_cuda kinect2_bridge.launch.py
ros2 launch bluelily_bridge bluelily_imu.launch.py

# Record (60 seconds, move checkerboard through all 6DOF)
./scripts/record_kalibr_bag.sh 60
```

### 4. Run Kalibr
```bash
# Convert bag to ROS1 (if using Docker Kalibr)
rosbags-convert calibration_results/kalibr_calib_<timestamp>

# Run calibration
cd kalibr_configs
kalibr_calibrate_imu_camera \
  --bag ../calibration_results/kalibr_calib_<timestamp>_ros1/kalibr_calib_<timestamp>_ros1_0.db3 \
  --cam kinect2_cam.yaml \
  --imu bluelily_imu.yaml \
  --target target.yaml \
  --show-extraction \
  --verbose
```

### 5. Extract Results
```bash
# Convert T_cam_imu to TF2 transform
./scripts/kalibr_to_tf2.py kalibr_configs/camchain-imucam-<timestamp>.yaml

# Copy the output and update bluelily_imu.launch.py
# Then rebuild and test
```

## Troubleshooting

### Kalibr can't detect corners
- Check lighting (even, no shadows)
- Verify target.yaml dimensions match printed board
- Ensure board is flat (no warping)
- Move slower during recording

### High reprojection error (>1.5 pixels)
- Re-measure checkerboard spacing with calipers
- Ensure board is perfectly flat
- Record new bag with more varied motion
- Check intrinsics are correct

### Optimization doesn't converge
- Verify IMU publishing at correct rate (100-800 Hz)
- Check camera publishing at ~14.5 Hz
- Include more fast rotations in recording
- Adjust timeshift_cam_imu parameter
